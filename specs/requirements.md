# Finwise Requirements and Project structure

## Workflow 
เป็น LINE OA เเละ LINE Mini App ที่สาารถสร้าง UI shortcut ได้ โดยมีการเชื่อมต่อกับระบบ backend เพื่อจัดการข้อมูลและฟังก์ชันต่างๆ ของแอปพลิเคชัน เเละมีความสามารถทำงานได้ใน Offline mode โดยการเก็บข้อมูลไว้ใน local storage หรือ IndexedDB เพื่อให้ผู้ใช้สามารถเข้าถึงข้อมูลและฟังก์ชันต่างๆ ได้แม้ไม่มีการเชื่อมต่ออินเทอร์เน็ต

เมื่อเปิดหน้า chat line จะสามารถคุยกับ bot ได้ เเละสามารถส่งค่าใช้จ่ายต่างๆ เช่น ค่าน้ำ ค่าไฟ ค่าโทรศัพท์ หรือค่าใช้จ่ายอื่นๆ ที่ต้องการติดตามได้ โดยการส่งข้อความในรูปแบบที่กำหนด เช่น "บันทึกค่าใช้จ่าย 500 บาท สำหรับค่าน้ำ" หรือ "บันทึกค่าใช้จ่าย 1000 บาท สำหรับค่าไฟ" เป็นต้น ระบบจะทำการประมวลผลข้อความที่ได้รับและบันทึกข้อมูลค่าใช้จ่ายลงในฐานข้อมูลของแอปพลิเคชัน นอกจากนี้ยังสามารถให้ผู้ใช้ดูรายงานค่าใช้จ่ายต่างๆ ได้ เช่น "ดูรายงานค่าใช้จ่ายเดือนนี้" หรือ "ดูรายงานค่าใช้จ่ายทั้งหมด" เป็นต้น ระบบจะทำการดึงข้อมูลจากฐานข้อมูลและแสดงผลในรูปแบบที่เข้าใจง่าย เช่น กราฟหรือรายงานสรุปค่าใช้จ่ายต่างๆ เพื่อให้ผู้ใช้สามารถติดตามและวิเคราะห์การใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ 


๊Use case 1.เปิด Line Mini App จะเป็นหน้า UI ในการอัปโหลดใบเสร็จรับเงินต่างๆ โดยผู้ใช้สามารถถ่ายรูปใบเสร็จรับเงินหรือเลือกจากแกลเลอรีของโทรศัพท์มือถือได้ หลังจากนั้นระบบจะทำการประมวลผลภาพใบเสร็จรับเงินเพื่อดึงข้อมูลที่สำคัญ เช่น ชื่อร้านค้า วันที่ซื้อสินค้า รายการสินค้า และจำนวนเงินที่ใช้จ่าย จากนั้นข้อมูลเหล่านี้จะถูกบันทึกลงในฐานข้อมูลของแอปพลิเคชันเพื่อให้ผู้ใช้สามารถติดตามและจัดการค่าใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ นอกจากนี้ยังสามารถให้ผู้ใช้ดูรายงานค่าใช้จ่ายต่างๆ ได้ เช่น "ดูรายงานค่าใช้จ่ายเดือนนี้" หรือ "ดูรายงานค่าใช้จ่ายทั้งหมด" เป็นต้น ระบบจะทำการดึงข้อมูลจากฐานข้อมูลและแสดงผลในรูปแบบที่เข้าใจง่าย เช่น กราฟหรือรายงานสรุปค่าใช้จ่ายต่างๆ เพื่อให้ผู้ใช้สามารถติดตามและวิเคราะห์การใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ

Use case 2.เป็น Line chat เเล้วสามารถคุยกับ bot ได้ เเละสามารถส่งค่าใช้จ่ายต่างๆ เช่น ค่าน้ำ ค่าไฟ ค่าโทรศัพท์ หรือค่าใช้จ่ายอื่นๆ ที่ต้องการติดตามได้ โดยการส่งข้อความในรูปแบบที่กำหนด เช่น "บันทึกค่าใช้จ่าย 500 บาท สำหรับค่าน้ำ" หรือ "บันทึกค่าใช้จ่าย 1000 บาท สำหรับค่าไฟ" เป็นต้น ระบบจะทำการประมวลผลข้อความที่ได้รับและบันทึกข้อมูลค่าใช้จ่ายลงในฐานข้อมูลของแอปพลิเคชัน นอกจากนี้ยังสามารถให้ผู้ใช้ดูรายงานค่าใช้จ่ายต่างๆ ได้ เช่น "ดูรายงานค่าใช้จ่ายเดือนนี้" หรือ "ดูรายงานค่าใช้จ่ายทั้งหมด" เป็นต้น ระบบจะทำการดึงข้อมูลจากฐานข้อมูลและแสดงผลในรูปแบบที่เข้าใจง่าย เช่น กราฟหรือรายงานสรุปค่าใช้จ่ายต่างๆ เพื่อให้ผู้ใช้สามารถติดตามและวิเคราะห์การใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ

ระบบจะเก็บข้อมูลรูปภาพของ Slip ไว้ใน Supabase Storage และข้อมูลอื่นๆ เช่น ชื่อร้านค้า วันที่ซื้อสินค้า รายการสินค้า และจำนวนเงินที่ใช้จ่าย จะถูกเก็บไว้ในฐานข้อมูลของ Supabase โดยใช้ Drizzle ORM เพื่อจัดการกับฐานข้อมูลและทำให้การเข้าถึงข้อมูลเป็นไปอย่างมีประสิทธิภาพ นอกจากนี้ยังมีการใช้ Docker เพื่อสร้างสภาพแวดล้อมการพัฒนาและการทดสอบที่เหมาะสม และ GitHub Actions เพื่อทำการ CI/CD ในการพัฒนาและปรับปรุงแอปพลิเคชันอย่างต่อเนื่อง

ความพิเศษคือต้องทำการ Train AI เพื่อให้สามารถประมวลผลภาพใบเสร็จรับเงินได้อย่างแม่นยำ โดยใช้เทคโนโลยีการเรียนรู้ของเครื่อง โดยใช้ Foundation Models ของ Openrouter มา train โดยใช้ ข้อมูลใบเสร็จรับเงินที่เก็บไว้ในฐานข้อมูล เพื่อให้ระบบสามารถดึงข้อมูลที่สำคัญจากภาพใบเสร็จรับเงินได้อย่างถูกต้องและมีประสิทธิภาพ นอกจากนี้ยังต้องมีการพัฒนาและปรับปรุงโมเดล AI อย่างต่อเนื่องเพื่อให้สามารถรองรับรูปแบบของใบเสร็จรับเงินที่หลากหลายและมีความซับซ้อนมากขึ้นในอนาคต

## Requirements
- Layout และ UI ที่สวยงาม ใช้งานง่าย และตอบสนองได้ดีบนอุปกรณ์มือถือเป็นหลัก
- การเชื่อมต่อกับ LINE Login เพื่อให้ผู้ใช้สามารถเข้าสู่ระบบด้วยบัญชี LINE ของตนเองได้อย่างง่ายดาย
- การจัดการข้อมูลค่าใช้จ่ายและใบเสร็จรับเงินอย่างมีประสิทธิภาพ โดยใช้ฐานข้อมูลที่เหมาะสมและมีความปลอดภัย
- การประมวลผลภาพใบเส็จรับเงินด้วย AI เพื่อดึงข้อมูลที่สำคัญและบันทึกลงในฐานข้อมูลอย่างถูกต้อง
- การแสดงรายงานค่าใช้จ่ายต่างๆ ในรูปแบบที่เข้าใจง่าย เช่น กราฟหรือรายงานสรุป เพื่อให้ผู้ใช้สามารถติดตามและวิเคราะห์การใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ
- การทำงานใน Offline mode เพื่อให้ผู้ใช้สามารถเข้าถึงข้อมูลและฟังก์ชันต่างๆ ได้แม้ไม่มีการเชื่อมต่ออินเทอร์เน็ต โดยการเก็บข้อมูลไว้ใน local storage หรือ IndexedDB และซิงค์ข้อมูลกับฐานข้อมูลเมื่อมีการเชื่อมต่ออินเทอร์เน็ตอีกครั้ง
- Chart เเสดงผลได้อย่างสวยงาม เเละเข้าใจง่าย โดยใช้ library ที่เหมาะสม เช่น Chart.js หรือ Recharts เพื่อให้ผู้ใช้สามารถวิเคราะห์ข้อมูลค่าใช้จ่ายของตนเองได้อย่างมีประสิทธิภาพ


## Tach stack
- Next.js 16
- Tailwind CSS
- Shadcn UI
- TypeScript
- Drizzle ORM
- Supabase
- Better Auth 
- LINE Login
- Docker
- GitHub Actions
- Openrouter Foundation Models (for AI training)
- Backend use Nextjs Route handler.
- Use typesafe API with Elysia (https://elysiajs.com/integrations/nextjs.html)


### Elysia
[Resource NextJS integration](https://elysiajs.com/integrations/nextjs.html) ใช้งานแบบ API Typesafe โดยใช้ TypeScript ในการพัฒนา API ซึ่งช่วยให้การพัฒนาและการบำรุงรักษา API เป็นไปอย่างมีประสิทธิภาพและปลอดภัย นอกจากนี้ยังมีความสามารถในการจัดการกับข้อมูลและการเชื่อมต่อกับฐานข้อมูลได้อย่างง่ายดาย ทำให้เป็นทางเลือกที่ดีสำหรับการพัฒนา API ในแอปพลิเคชัน Next.js โดยเฉพาะอย่างยิ่งเมื่อใช้ร่วมกับ Drizzle ORM และ Supabase เพื่อจัดการกับฐานข้อมูลและการเก็บข้อมูลรูปภาพของ Slip ใน Supabase Storage ได้อย่างมีประสิทธิภาพและปลอดภัย

## UI Library 
- Shadcn UI: A collection of accessible and customizable UI components built on top of Tailwind CSS. It provides a wide range of pre-designed components that can be easily integrated into the Next.js application, ensuring a consistent and visually appealing user interface. Important UI components should be reusable and customizable to fit the specific needs of the application. This will allow for efficient development and maintenance of the UI while ensuring a cohesive design throughout the application.
